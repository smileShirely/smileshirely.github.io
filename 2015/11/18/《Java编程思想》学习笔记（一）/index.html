从Java编程思想中取学习

Java基础知识

  • 为了给一个类库生成不会与其他名字混淆的名字，Java设计者希望程序员反过来使用自己的Internet域名，以保证它们呢是独一无二的
  • 为了消除类名容易混淆的冲突，可以用关键字import来准确地告诉编译器你想要的类是什么。import指示编译器导入一个包，就 是一个类库。大对数时候，我们使用与编译器附在一起的Java类库里的构件，就只需要import java.util.ArrayList:
    这行代码告诉了编译器，你想使用Java的ArrayList类。可以用通配符 来达到这个目的：import java.util.*
  • static关键字：当声明一个事物是static时，就意味着这个域或方法不会与包含它的那个类的任何对象关联在一起

    开始写Java程序

    我是用的Eclipse，网上教程很多，大家可以了解一下怎么在这个编译环境下写Java程序。

打印出来的结果就是

Hello,it’s:
Wed Nov 18 15:36:25 CST 2015

运行的快捷键是:ctrl + F11
我们来分析一下这句话， public static void main(String[] args) {
其中，public关键字指的是一个可由外部调用的方法，main()方法的参数是一个String对象的数组。在这个程序中并未用到Java编译器要求必须这样做，因为args要用来存储命令行参数。args是一个字符串数组的变量名，不是关键字，是arguments的缩写

书上提到了javadoc

序源代码中抽取类、方法、成员等注释形成一个和源代码配套的API帮助文档。也就是说，只要在编写程序时以一套特定的标签作注释，在程序编写完成后，通过Javadoc就可以同时形成程序的开发文档了。
javadoc命令是用来生成自己API文档的，使用方式：在dos中在目标文件所在目录输入javadoc +文件名.java。

觉得很好奇，就自己百度了一下，<http://jingyan.baidu.com/article/2c8c281df777d90008252aff.html>，跟着这个教程，给刚刚生成的java程序，快速生成了javadoc，顿时对java学习来了兴趣。也参考了一下<http://blog.csdn.net/shy_snow/article/details/6501623>，在Eclipse中也生成了javadoc。

关于编码风格

  • 因为在”Java编程语言编码约定”中，代码风格是规定的是类名的首字母要大写；如果类名由几个单词构成，就把它们并在一起（也就是说，不要用用下划线来分割名字，PS：虽然我平时老是爱这样），其中每个内部单词的首字母都采用大写的形式。这种风格也称为“驼峰风格”（其实最开始知道这个名词，还是同为程序员的男票告诉我的）。
    这个风格几乎适用于java中的其他所有内用，除了标识符以外（标识符的第一个字母采用小写）
